# 项目文件说明

## 📁 核心配置文件

### `.gitlab-ci.yml` ⭐ **最重要**
**作用：** GitLab 自动部署配置文件
- 告诉 GitLab 如何部署你的网站
- 当你推送代码时，GitLab 会自动运行这个文件中的命令
- 会把所有 HTML 文件复制到 `public` 文件夹，然后部署到 GitLab Pages

**内容解释：**
```yaml
pages:              # 这是 GitLab Pages 的部署任务
  script:           # 执行的命令
    - mkdir -p public                    # 创建 public 文件夹
    - cp -r *.html public/              # 复制所有 HTML 文件
    - cp -r lab guide tool... public/   # 复制各个目录
  artifacts:
    paths:
      - public      # 告诉 GitLab 部署 public 文件夹的内容
```

### `.gitignore`
**作用：** 告诉 Git 哪些文件不需要上传
- 比如临时文件、系统文件等
- 这些文件不会被推送到 GitLab

### `robots.txt`
**作用：** 告诉搜索引擎哪些页面可以抓取
- 帮助 Google 等搜索引擎索引你的网站
- 包含 sitemap 地址

### `sitemap.xml`
**作用：** 网站地图，告诉搜索引擎有哪些页面
- 列出所有重要页面的地址
- 帮助搜索引擎更快发现和索引你的网站

## 🌐 网站文件

### `index.html`
**作用：** 网站首页
- 访问网站时看到的第一个页面

### `guide/` 目录
**作用：** 指南页面
- 包含所有学习指南的 HTML 文件
- 比如：快速入门、框架详解等

### `tool/` 目录
**作用：** 工具页面
- 介绍各种工具的使用方法

### `compare/` 目录
**作用：** 对比页面
- 不同方法、工具的对比分析

### `lab/` 目录
**作用：** 实验区页面
- 探索性内容，可以随时删除

### `live/` 目录
**作用：** 实战案例页面
- 实际项目案例和演示

### `assets/css/main.css`
**作用：** 网站样式文件
- 控制网站的外观、颜色、布局等

## 🎯 关键理解

1. **`.gitlab-ci.yml`** 是最重要的文件，它告诉 GitLab 如何部署
