# 🌐 如何测试网站是否正常

## 方法一：本地测试（现在就可以测试）

### 1. 打开浏览器
打开你常用的浏览器（Chrome、Safari、Firefox 等）

### 2. 访问本地地址
在浏览器地址栏输入：
```
http://localhost:8000
```

### 3. 测试各个页面
点击导航栏的链接，测试以下页面：
- ✅ 首页：`http://localhost:8000/`
- ✅ 指南：`http://localhost:8000/guide/`
- ✅ 工具：`http://localhost:8000/tool/`
- ✅ 对比：`http://localhost:8000/compare/`
- ✅ 实战：`http://localhost:8000/live/`
- ✅ 实验区：`http://localhost:8000/lab/`

### 4. 检查内容
- [ ] 页面能正常显示
- [ ] 导航链接可以点击
- [ ] 样式正常（颜色、布局）
- [ ] 内容完整显示
- [ ] 没有 404 错误

### 5. 停止本地服务器
测试完成后，在终端按 `Ctrl + C` 停止服务器

---

## 方法二：在线测试（推送到 GitLab 后）

### 1. 推送到 GitLab
按照 `快速开始.md` 的步骤推送代码

### 2. 等待部署完成
- 在 GitLab 项目页面查看 CI/CD 状态
- 等待变成绿色 ✅（通常 3-5 分钟）

### 3. 获取网站地址
在 GitLab 项目页面：
- 点击 **Settings** → **Pages**
- 复制显示的网站地址（类似：`https://你的用户名.gitlab.io/pseoexample/`）

### 4. 在浏览器访问
直接打开上面的地址，测试网站

## ❗ 常见问题

### Q: 本地测试显示 404
**解决：** 确保本地服务器正在运行
```bash
# 重新启动服务器
cd /Users/test/pseoexample
python3 -m http.server 8000
```

### Q: 样式没有加载（页面很丑）
**解决：** 检查 CSS 文件路径
- 确认 `assets/css/main.css` 文件存在
- 检查浏览器控制台是否有错误

### Q: 链接点击后显示 404
**解决：** 检查文件路径是否正确
- 确认所有 HTML 文件都在对应目录下
- 检查文件名是否匹配

## 🎯 快速测试命令

在终端运行以下命令快速检查：

```bash
# 检查首页
curl -s http://localhost:8000 | grep -o "<title>.*</title>"

# 检查 CSS
curl -s -o /dev/null -w "%{http_code}" http://localhost:8000/assets/css/main.css

# 检查指南页面
curl -s http://localhost:8000/guide/ | grep -o "<title>.*</title>"
```

如果都返回正常，说明网站运行正常！
