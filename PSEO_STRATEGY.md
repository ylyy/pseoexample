# PSEO 策略实施说明

## 项目结构

本项目严格按照 NICHEDIGGER PSEO 策略构建，实现了角色隔离和内链权重流向控制。

### 目录角色

```
/
├── lab/          # 实验区（短期曝光，可丢弃）
│   └── 探索性内容，生命周期预设
│
├── guide/        # 指南区（长期转化，决策内容）
│   └── 核心学习内容，高信任度
│
├── tool/         # 工具区（辅助内容）
│   └── 开发工具和资源
│
├── compare/      # 对比区（强商业意图）
│   └── 框架和工具对比
│
└── live/         # 实战区（实际案例）
    └── 项目演示和案例
```

## 内链策略

### 单向漏斗结构

```
lab (实验区)
  ↓
guide / tool (指南/工具)
  ↓
compare / live (对比/实战)
```

### 内链规则

1. **lab → guide/tool**: 实验区内容可以链接到指南和工具页
2. **guide/tool → compare/live**: 指南和工具页链接到对比和实战页
3. **禁止反向链接**: compare/live 不链回 lab，避免权重流失

### Anchor Text 策略

- **lab → guide**: 使用精确/半精确匹配（如 "React 入门指南"）
- **guide → compare**: 使用商业意图词（如 "框架对比"、"选择最适合"）
- **tool → guide**: 使用功能描述（如 "了解更多"、"学习指南"）
